Interview Experience at Druva

Himanshu

about druva:

The company provides SaaS-based data protection and management products. It has roles for both SDET and SDE, kkwagh is usually approached for the SDET role.

tech stack for:
SDET - ( package offered: 18 LPA )
involves the robot framework, selenium and api automation.
SDE - ( package offered: 20-25 LPA )
involves python and go , mostly involved with python. requires applied skills for bfs , dfs , hashmaps and a good hold of optimization algorithms.

recruitment process:

- coding round (online)
- technical round 1
- technical round 2


coding round:

had 3 coding questions and 10 mcqs
- string challenge
: involved reading an input string and performing operations accordingly
- math challenge
: Math operations
- array challenge
: Array manipulation


technical round 1

was asked to screenshare and code according to the instructions given. The challenge given to me was string compression - for a string aaabbccddd the output should be a3b2c2d3 they ask you what approach you're going to take for the problem and you can explain it to them as you write the code. after the code was complete , i was asked to write test cases for the given code. all the edge cases are to be covered.


technical round 2

this involves logical reasoning and technical theory. I was asked theory questions on subjects such as OS (operating systems) , CNS (computer networks) ,DSA, DBMS. They further ask pratical questions to test applied knowledge on these topics. Next , they ask coding questions that you are expected to only explain the approach for. you have to explain how you'll solve the given problem and also the different ways you'll optimize it. These are very logical in nature and don't require you to code or provide any pseudo code to the interviewer.


post interview:

i was approached by the HR to inform that i was selected for the SDET role for a package of 18 LPA. after starting the training period in the company, within one month i was approached by my manager to inquire about my interest in the developer side based on my past performance in the company. further interview was conducted internally within the company which involved solving (coding) dsa questions ( same level as leetcode medium ) , optimising them and explaining the approach. further questions were asked on my api based projects. within a week i was offered an internal role switch within druva from SDET to SDE
